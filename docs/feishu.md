二、前置UI设计
叙境 (故宫 Demo) UI 前置设计文档 v1.0
1. 设计哲学：数字化的宫廷手记
- 设计隐喻： 用户持有一本“随行的数字游记”。
- 视觉基调： 东方美学中的“低对比度沉浸”。拒绝现代 UI 常见的纯白与高饱和色，采用纸张的天然暖白与沉积已久的墨色，辅以朱砂红作为关键行为触发器。
- 交互原则： “触碰—推拉—揭示”。所有 UI 元素应随交互产生微弱的动态，模拟物理世界的阻力与重量。

2.2 字体体系 (Typography)
- Display (标题)： LXGW WenKai (霞鹜文楷)。用于宫门、章节名、关键叙事锚点。
- Body (正文)： PingFang SC / Noto Sans SC。用于操作说明、AI 生成的话术。
- 字号建议： 标题 24px-28px，正文 16px，说明 Caption 13px。保证阳光下可读性。
3. 开篇交互设计规范 (Key Screens)
3.1 序列一：入宫仪式 (The Entry Screen)
- 背景： 全屏朱红大门，采用高精度木纹与漆面材质（Bump Map 质感）。
- 交互项：
  - 铜铺首： 位于屏幕正中心，设计为 3D 浮雕效果。
  - 动作控制： 绑定 TouchMove 事件，用户向下拖拽铺首。
  - 反馈动效： 伴随拖拽位移，大门产生 scale 与 perspective 偏移，模拟门的厚重感。
  - 音效： 加载 door_creak.mp3。
3.2 序列二：光影过渡 (Atmospheric Transition)
- 视觉： 门缝展开后，不直接显示内容。通过 mask-image 实现由内向外的光束（丁达尔效应），光束中动态平移微小的颗粒（灰尘动效）。
- 文案展示： 随着光束移动，一行淡墨色小字缓慢浮现（opacity: 0->1, letter-spacing: 0.1em）。
3.3 序列三：悬念触发 (The Trigger)
- 交互： 画面聚焦于地面，一块带有裂纹的古砖作为触发点。
- 视觉： 裂纹处通过 filter: drop-shadow(0 0 10px #B8923A) 呈现微弱的金光。
- 反馈： 用户点击后，触发界面卷轴式展开动效（transform: scaleY(0->1), transform-origin: top），无缝连接至剧情线。
4. UI 动效原则 (Motion Design)
- 速度曲线： 绝不使用 linear，统一采用 cubic-bezier(0.22, 1, 0.36, 1)。
- 质感反馈：
  - 按钮点击：scale(0.98) + 极淡的朱砂色波纹扩散。
  - 页面切换：模拟卷轴滚动的视觉，上下出入，严禁左右切换。
- 零侵入设计： 屏幕顶部不要放任何状态栏颜色，建议使用 transparent 或与 paper 颜色统一。
5. 开发建议 (Engineering Notes)
- 兼容性： 确保 H5 页面在 iPhone 微信容器中的 viewport-fit=cover 处理。
- 性能优化：
  - 大门木纹与铺首素材使用 WebP 格式，确保在低速网络下 2 秒内加载完毕。
  - 粒子动效（光束灰尘）使用轻量级 Canvas 渲染，不要使用大 GIF，防止卡顿。
- 触感整合： 必须调用手机震动 API (Haptic Feedback)，在拉动铺首的那一刻给出 15ms 的短震动，这比视觉更能提升高级感。

- [ ] 是否存在任何现代 UI 常用的“玻璃拟态（毛玻璃）”效果？（如果存在，请移除，改为宣纸底色）

三、打卡勋章和打卡具体形式
1. 打卡具体形式：模拟“文博动作” (Interaction)
游客不再点击“打卡按钮”，而是通过交互动作完成一次“数字化的文物修复”。
- 交互触发： 当用户通过 GPS 或近场校验进入冷门展馆后，手机屏幕自动从当前的剧情对话页，无缝平滑过渡到“修复交互页”。
- 物理动作设计：
  - 第一步：清灰 (Clean) —— 屏幕覆盖着一层细腻的“数字尘埃”，用户需要用指尖在屏幕上做擦拭动作。随着擦拭，屏幕下方浮现出展品的局部纹理。
  - 第二步：补色 (Infill) —— 屏幕中央出现一件带有残缺的“青铜器”或“瓷器”影像，系统提示“在此处点击以重现色彩”。用户点击后，色泽如墨水般蔓延，整件器物变得完整且色泽明亮。
  - 第三步：落印 (Stamp) —— 屏幕中央出现一枚待盖的御玺，用户需要长按印章 1.5 秒（模拟盖章时的下压手感），伴随金属撞击声与红泥溢出的动效，标志着本次打卡的完成。
2. 打卡勋章
勋章设计必须是“完整、璀璨、具有高级金属质感”的。
- 勋章形态： 每一枚勋章都是一枚“立体的 3D 掐丝珐琅金印”。
  - 设计细节： 勋章中央为该展馆代表性文物的抽象纹样（如延禧宫的鸟纹，珍宝馆的龙纹），四周由金丝构成的云雷纹缠绕。
  - 动态呈现： 在“多宝格”中，这些勋章带有金属光泽的物理渲染 (PBR)，当你晃动手机，光影会在金丝与珐琅面上流转。
- 获得感： 当用户完成上述“清灰+补色+落印”的动作后，勋章会从屏幕中心缓缓升起，随后像磁铁一样“吸入”到你的“多宝格”中，格子周围泛起一圈淡淡的“流光”。
3. 多宝格足迹墙：用户的“数字藏宝库” (The Wall of Footprints)
这将是用户最愿意分享的界面。
- 布局： 采用“九宫格多宝格”布局，深红木质纹理背景，每个格子里嵌入不同大小的“御制金印”。
- 动态： 随着用户打卡数量增加，空的格子会被金色的光亮填补。整个墙面支持 陀螺仪重力感应 —— 你倾斜手机，勋章表面的光影也会随之改变，展示出极其昂贵的质感。
- 分享页面 (Social Share)：
  - 点击“分享我的藏宝阁”，生成一张“故宫私人馆藏展海报”。
  - 海报主体是用户已点亮的勋章阵列，下方配文：“我在故宫完成了 XX 处历史碎片的溯源修复，这是我留给紫禁城的数字印记。”
四、预生成模版
1. 模版设计思路：三层架构，以卡片形式呈现
1.1 全局模板（逻辑层）： 规定故事的走向（比如：引入悬疑 -> 给出线索 -> 利益引导）。
1.2 宫殿信息库（知识层）： 这是一个简单的 Excel，列出：宫殿名、核心文物、独特建筑特征、对应的推荐跳转点。
1.3 动态拼装（AI 辅助层）： 你的系统在用户扫码时，自动把“宫殿信息”填入“全局模板”。

2. 具体文稿 
 “叙境·三步法”模板。
场景设定：游客在【当前馆】（拥挤点），我们要引导他去【跳转馆】（冷门点）。
【开场：认知重构】
- “这里的 [当前核心文物/现象] 其实并不是它最真实的样子。你看它背后的 [独特建筑特征/工艺细节]，在历史上其实指向了一个被遗忘的角落。”
【线索：疑点植入】
- “那里的 [某件关联文物/古籍/印章] 才是揭开这个谜底的钥匙。如果你能找到它，你就能补全这整段故宫密闻。”
【引导：利益驱动】
- “既然现在这里人潮拥挤，不如先去 [跳转馆] 寻回那段历史。完成后回馈给你一份 [专属勋章/文创折扣]，作为你守护历史碎片的奖励。”

3. 实操演示
策略：不需要每个宫都设： 只挑选 3 个流量高压区 作为“出发点”，3 个流量低谷区 作为“引导目的地”。一共只需要维护 9 组碎片信息（即 3*3 的连线组合）。
案例 1：钟表馆 -> 延禧宫
- 填空：
  - 当前馆：钟表馆（人多）
  - 独特特征：写字人钟的机械传动
  - 跳转馆：延禧宫
  - 利益：专属御制金印
- 生成的文稿：
- “这写字人钟的齿轮传动，其实并不是中国工匠的独创。你看它底座的纹路，与延禧宫当年的水殿遗存如出一辙。去延禧宫，那里的残图能揭开这段中西合璧的工法秘密。若你寻回了线索，可获得延禧宫御制金印一枚。”

案例 2：珍宝馆 -> 寿康宫
- 填空：
  - 当前馆：珍宝馆（人多）
  - 独特特征：清宫后妃的珐琅工艺
  - 跳转馆：寿康宫
  - 利益：宫廷印记
- 生成的文稿：
- “这些金银细工在珍宝馆里显得过于冰冷，因为它们缺少了当年的生活温度。寿康宫里藏着当年后妃们手制的珐琅小件，那里才是这些珍宝原本的归宿。去寻回它们的温度吧，那是开启寿康宫宫廷印记的唯一线索。”
