# 叙境 DESIGN.md

## 1. Visual Theme and Atmosphere

东方人文纸质质感，以"手持会讲故事的景区游记"为体验隐喻。整体轻量、通透，适合户外边走边看。拒绝数字冷感和模板化卡片堆叠，用留白、线装书般的细线分隔、以及卷轴式展开动效传递叙事温度。密度适中——信息足够完成当前任务，但不拥挤。

## 2. Color Palette and Roles

主色调采用"宣纸-墨色-朱砂"三色体系，辅以竹青用于状态区分。所有色值使用 HEX，适配移动端 H5 CSS 环境。

| Token | Value | Role |
|---|---|---|
| `paper` | `#F7F4ED` | 页面底色，模拟宣纸暖白 |
| `paper-deep` | `#EFEBE1` | 卷轴展开区底色、卡片底层 |
| `ink` | `#2B2926` | 主标题、正文，偏暖的墨色（非纯黑） |
| `ink-light` | `#6B6860` | 次要文字、说明文字 |
| `ink-faint` | `#A09B90` | 占位符、禁用状态、时间戳 |
| `cinnabar` | `#C13A3A` | 主强调色：朱砂红，用于主按钮、进度节点、解锁提示 |
| `cinnabar-light` | `#F5E3E3` | 朱砂的极淡底色，用于选中态背景、徽章底色 |
| `bamboo` | `#5A7D5A` | 辅助色：竹青，用于成功打卡、已解锁、正向状态 |
| `bamboo-light` | `#E8F0E8` | 竹青淡底，用于成功态背景 |
| `gold` | `#B8923A` | 点缀色：暗金，用于纪念徽章、贵重奖励、特殊边框 |
| `scroll-line` | `#D4CFC3` | 卷轴分隔线、细边框，比背景深一个色阶 |
| `shadow` | `rgba(43, 41, 38, 0.06)` | 极淡投影，仅用一层，拒绝多层阴影 |

**主题选择**：本方案为 light-mode 明确设计。H5 页面在户外通过手机浏览器使用，浅色底配合高对比墨色正文，保证阳光下可读性。深色模式暂不强制，但所有颜色基于语义 token，后续可扩展。

## 3. Typography Rules

字体策略：通过 `@font-face` 加载霞鹜文楷（LXGW WenKai）作为 Display 字体，正文使用系统字体栈确保性能与可读性。基于 375px 移动端视口定义尺寸。

```css
@font-face {
  font-family: 'LXGWWenKai';
  src: url('https://cdn.jsdelivr.net/npm/lxgw-wenkai-webfont@1.7.0/lxgwwenkai-regular.woff2') format('woff2');
  font-weight: 400;
  font-display: swap;
}
```

| Role | Font | Size | Weight | Line-height | Letter-spacing |
|---|---|---|---|---|---|
| Display | LXGW WenKai, PingFang SC | 28px | 400 | 1.3 | 0.04em |
| H1 | LXGW WenKai, PingFang SC | 24px | 400 | 1.35 | 0.03em |
| H2 | LXGW WenKai, PingFang SC | 22px | 400 | 1.4 | 0.02em |
| H3 | PingFang SC, Noto Sans SC | 18px | 600 | 1.45 | 0 |
| Body | PingFang SC, Noto Sans SC | 16px | 400 | 1.65 | 0.01em |
| Caption | PingFang SC, Noto Sans SC | 13px | 400 | 1.5 | 0.02em |
| Tag | PingFang SC, Noto Sans SC | 12px | 500 | 1.4 | 0.04em |

**规则**：
- Display 和标题使用霞鹜文楷，传递人文书法气息；正文使用系统无衬线，保证长时间阅读舒适度。
- 大字号（18px 以上）使用正字间距或微正间距，不压缩，因为中文压缩会降低辨识度。
- 仅对数字和计数器使用 `font-variant-numeric: tabular-nums`（CSS 中直接支持）。

## 4. Component Stylings

### 主按钮（Primary Action）
- 背景：`cinnabar`
- 文字：`paper`，16px，Weight 500
- 圆角：`radius-pill`（完全圆角，即 9999px）
- 高度：48px，最小触控区域满足 40px
- 状态：
  - Hover（`@media(hover:hover)`）：亮度提升 5%（`#D44A4A`）
  - Active：`scale(0.96)`，transition 150ms ease-out
  - Disabled：背景 `ink-faint`，文字 `paper`

### 次按钮（Secondary Action）
- 背景：透明
- 边框：1px solid `cinnabar`
- 文字：`cinnabar`，16px
- 圆角：`radius-pill`
- Active：`scale(0.96)`

### 卷轴卡片（Scroll Card）
- 背景：`paper-deep`
- 圆角：`radius-lg`（8px）
- 内边距：20px
- 不使用阴影，使用上下 `scroll-line` 细线边框营造卷轴感
- 特殊状态（解锁时）：顶部加 2px `gold` 实线

### 兴趣标签（Interest Tag）
- 默认：背景 `paper-deep`，边框 1px solid `scroll-line`，文字 `ink-light`
- 选中：背景 `cinnabar-light`，边框 1px solid `cinnabar`，文字 `cinnabar`
- 圆角：`radius-pill`
- 内边距：8px 16px
- Transition：background-color 200ms, border-color 200ms, color 200ms

### 进度节点（Progress Node）
- 已完成：实心圆，背景 `bamboo`，大小 10px
- 当前：实心圆，背景 `cinnabar`，大小 12px，外围 1px 同色圆环
- 未完成：空心圆，边框 1px solid `scroll-line`，大小 10px
- 连接线：1px solid `scroll-line`（未完成段用虚线）

### 输入框（Input Field）
- 背景：`paper`
- 边框：底部 1px solid `scroll-line`（仅底边，非全包围）
- 聚焦：底部变为 `cinnabar`
- 文字：`ink`，16px
- Placeholder：`ink-faint`

## 5. Layout Principles

- **视口适配**：页面必须包含 `<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no">`。
- **边距体系**：页面左右统一 20px 内边距，垂直元素间距以 16px 为基准单位（1 step）。
- **内容宽度**：移动端单列流式布局，内容区最大宽度 100%，在宽屏设备上居中显示且不超过 480px。
- **卷轴隐喻**：关键内容区（如任务卡、剧情文本）不使用标准卡片阴影，而是采用上下细线边框 + 略深背景色（`paper-deep`），模拟卷轴展开区域的边界。
- **留白策略**： generous whitespace。每屏只做一件事，拒绝信息堆叠。
- **对齐**：文字左对齐为主，按钮和短标题可居中。任务流以可用性优先。

## 6. Depth and Elevation

移动端 H5 环境下深度通过以下方式表达，拒绝玻璃拟态和多层阴影：

| Level | Treatment |
|---|---|
| 页面底层 | `paper` 纯色 |
| 卷轴展开区 | `paper-deep` + 上下 `scroll-line` 1px 边框 |
| 悬浮按钮/行动点 | `paper` 背景 + 单层阴影 `0 2px 8px rgba(43,41,38,0.08)` |
| 顶部导航 | `paper` 背景 + 底部 1px `scroll-line` 边框 |

**无玻璃拟态，无模糊背景**。所有表面为实色，深度通过色阶差异实现。

## 7. Do's and Don'ts

1. **Do** 使用卷轴上下细线边框替代卡片阴影。**Don't** 使用带圆角阴影的通用卡片模板。
2. **Do** 让霞鹜文楷只负责标题和简短文案（≤20字）。**Don't** 用它渲染长段落正文，影响阅读效率。
3. **Do** 朱砂红（`cinnabar`）只出现在主按钮、当前进度节点、解锁提示上，占页面色彩面积不超过 10%。**Don't** 把朱砂用作大面积背景。
4. **Do** 任务完成反馈使用竹青（`bamboo`）+ 印章/对勾图标。**Don't** 用绿色系做品牌主色，竹青仅用于状态。
5. **Do** 保持每屏一个核心行动（CTA）。**Don't** 在任务引导页同时放"去打卡"和"跳过任务"两个同等权重的按钮。
6. **Do** 图片使用 `border-radius: radius-md (6px)` 并加 inset outline `outline: 1px solid rgba(43,41,38,0.1); outline-offset: -1px`。**Don't** 让图片直接贴边或全屏出血无边界。
7. **Do** 所有动画使用 CSS transition（transform, opacity）。**Don't** 使用复杂 keyframe 动画或 `animation` 做花哨效果，避免低端机卡顿。
8. **Do** 打卡成功的页面进入动画用 400ms ease-out。**Don't** 使用 bounce 或 elastic 缓动。
9. **Do** 为所有 hover 状态包裹 `@media(hover:hover)`。**Don't** 让触摸设备上的元素保留永久 hover 状态。

## 8. Responsive Behavior

H5 页面运行在各类手机浏览器中，需适配不同屏幕尺寸和安全区：

- **视口**：强制 `width=device-width, initial-scale=1.0`，禁止用户缩放。
- **iPhone 安全区**：底部固定按钮区域增加 `env(safe-area-inset-bottom)` 的 padding-bottom。
- **全面屏适配**：顶部状态栏高度通过 CSS `padding-top: env(safe-area-inset-top)` 或固定 44px 留白处理。
- **字号适配**：所有尺寸使用 px，基于 375px 设计稿。在宽屏设备上（如平板）内容区最大宽度限制为 480px 并居中。
- **横屏**：通过 CSS `@media (orientation: landscape)` 提示用户竖屏使用，或保持单列布局自适应。
- **触控目标**：所有可交互元素最小触控区域 40×40px，小图标通过伪元素扩展热区。

### 组件 Prompt 示例

**Prompt 1: 兴趣选择页**
> 创建一个移动端 H5 页面，背景色 `#F7F4ED`，顶部居中标题"选择你的探索偏好"，使用 24px 霞鹜文楷（font-family: 'LXGWWenKai', 'PingFang SC', sans-serif），颜色 `#2B2926`。下方是 2x3 的网格兴趣标签：历史、建筑、人物、亲子、悬疑、工艺。标签默认态：背景 `#EFEBE1`，边框 1px `#D4CFC3`，文字 `#6B6860`，圆角 9999px，内边距 8px 16px。选中态：背景 `#F5E3E3`，边框 `#C13A3A`，文字 `#C13A3A`。标签切换 transition 200ms。页面底部一个全宽主按钮"开启旅程"，背景 `#C13A3A`，文字 `#F7F4ED`，高度 48px，圆角 9999px，active 态 scale(0.96)。页面进入动画：整体 opacity 0→1，translateY(20px)→0，400ms ease-out。页面需包含 viewport meta。

**Prompt 2: 剧情任务页**
> 创建一个移动端 H5 任务展示页面，背景 `#F7F4ED`。顶部进度条：3 个圆点节点，已完成节点背景 `#5A7D5A` 实心圆 10px，当前节点背景 `#C13A3A` 实心圆 12px 带 1px 同色外环，未完成节点空心圆边框 `#D4CFC3`。节点间连接线 1px `#D4CFC3`。页面中部是一个"卷轴卡片"区域：背景 `#EFEBE1`，圆角 8px，上下边框 1px `#D4CFC3`，内边距 20px。卡片内标题"密信的另一半"使用 22px 霞鹜文楷 `#2B2926`，正文内容 16px 系统字体 `#2B2926`，行高 1.65。卡片底部有一个"前往数字馆"主按钮，样式同 Prompt 1。页面进入时，卡片区域使用 translateY(12px)→0，opacity 0→1，400ms ease-out 的展开效果。

**Prompt 3: 打卡成功反馈页**
> 创建一个移动端 H5 打卡成功反馈页面，背景 `#F7F4ED`。页面中央是一个大型状态图标：圆形背景 `#E8F0E8`，直径 80px，居中一个对勾图标颜色 `#5A7D5A`。图标下方标题"打卡成功"，24px 霞鹜文楷 `#2B2926`。副标题"你找到了密信的第二条线索"，14px `#6B6860`。下方卷轴卡片展示解锁内容：如果解锁优惠券，卡片顶部加 2px `#B8923A` 实线，卡片内显示券码和商户信息。底部主按钮"继续探索"背景 `#C13A3A`。整个页面进入动画：图标先 scale(0.8)→scale(1) 配 opacity 0→1，耗时 300ms ease-out；文字延迟 100ms 后 translateY(8px)→0，opacity 0→1。

**Prompt 4: 运营后台概览卡片**
> 创建一个桌面端后台的数据概览卡片组件（如用于 Web 后台），背景 `#F7F4ED`，圆角 8px，不使用阴影，使用 1px `#D4CFC3` 边框。卡片顶部左侧一个 4px 宽、16px 高的 `#C13A3A` 竖条作为标识（拒绝宽 border-left）。卡片内标题 16px 系统字体 `#2B2926` weight 600，下方大数字 36px `#2B2926` weight 700，`font-variant-numeric: tabular-nums`。数字下方趋势标签：上升用 `#5A7D5A` 背景淡底，下降用 `#C13A3A` 背景淡底。

| P6 | **任务完成总览页** | 全部 3 段任务完成后展示纪念卡与路线回顾 |

### 10.2 页面流转

```
扫码进入 → [P1 入口页] → 点击"开启旅程"
              ↓
       [P2 兴趣选择页] → 选择标签 → 点击"生成剧情"
              ↓
       [P3 剧情任务页] ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ←
              ↓ 点击"前往 xxx"                              ↑
       [P4 打卡确认页] → 点击"确认到达"                      │
              ↓                                              │
       [P5 奖励解锁页] → 点击"继续探索" ─────────────────────┘
              ↓ (3 段全部完成)
       [P6 任务完成总览页]
```

游客在一次完整的 demo 体验中，会经历 **3 次"剧情任务 → 打卡确认 → 奖励解锁"的循环**。每次循环对应一组 3×3 连线组合中的一个跳转（如钟表馆 → 延禧宫）。

## 13. 与 3×3 场景的视觉映射

### 13.1 页面 → 业务场景的对应关系

| 页面 | 3×3 场景中的角色 |
|---|---|
| P1 入口页 | 统一入口，与具体点位无关 |
| P2 兴趣选择页 | 决定叙事风格（历史/悬疑/亲子等），影响后续 9 组文案的语气 |
| P3 剧情任务页 | **核心映射页**。根据游客当前所在馆（3 个出发点之一），展示对应的引导文案。系统从 9 组连线库中匹配当前馆+兴趣标签的文案 |
| P4 打卡确认页 | 游客到达 3 个目的地之一后确认 |
| P5 奖励解锁页 | 发放该目的地的专属奖励（如"延禧宫御制金印"） |
| P6 任务完成总览页 | 展示 3 段完整路线（如钟表馆→延禧宫→珍宝馆→寿康宫→陶瓷馆→慈宁宫） |

### 13.2 P3 剧情任务页的内容填充逻辑

系统根据两个维度决定 P3 页面展示的内容：

1. **当前馆**（出发点）：决定卡片标题和正文的前半段（"当前馆独特特征"）
2. **目的地**（由 Route Planner 根据拥堵和兴趣分配）：决定按钮文案、目的地提示、奖励名称

**示例**：游客当前在钟表馆，系统分配目的地为延禧宫，兴趣标签为"悬疑"

- 当前馆标识："当前：钟表馆"
- 卡片标题："齿轮里的秘密"
- 正文：悬疑版引导文案（见 PRD 1.1）
- 目的地提示："线索指向 → 延禧宫·灵沼轩"
- 按钮："前往延禧宫"
- 预期奖励（P5 展示）："延禧宫御制金印"

## 15. Do's and Don'ts（页面层面补充）

1. **Do** 剧情任务页的正文区域最多显示 6 行，超出时使用 `max-height + overflow-y: auto` 内部滚动，不滚整页。**Don't** 让长文案撑开整页导致按钮被顶到屏幕外。
2. **Do** 打卡确认页的"确认打卡"按钮在点击后进入 2 秒不可点击状态（防止重复提交），并显示加载动画。**Don't** 允许用户连续快速点击导致重复打卡。
3. **Do** 奖励解锁页的插画卡片使用 `aspect-ratio: 16/9` 保持比例。**Don't** 让插画因图片尺寸不同而变形。
4. **Do** 所有页面底部固定按钮区域统一使用 `padding-bottom: calc(20px + env(safe-area-inset-bottom))`。**Don't** 让按钮被 iPhone 底部横条遮挡。
5. **Do** 页面切换使用 `opacity + translateY` 的淡入效果，保持统一的转场节奏。**Don't** 使用左右滑动切页，容易与浏览器的返回手势冲突。
