📱 叙境 (Xujing) 交互逻辑全流程

第一阶段：系统初始化 (静默闭环)
扫码/进入 (确认出发点)：页面自动识别地理位置 Location_ID。
身份生成 (确认ID)：系统在后台静默分配 Session_ID，并记录游客选择的兴趣标签。
兴趣选择 (选择讲解风格)：弹出极简卡片选择兴趣标签：[历史] / [建筑] / [人物] / [亲子] / [悬疑] / [工艺]。该标签一旦选定，后续所有 Narrative 页面均基于此标签渲染单一叙事，不设多分支切换。

第二阶段：核心交互流 (秘辛地图 + 路线引导 + 实地解锁)
Explore · 秘辛地图 (选关中枢)：
  展示全部 6 个点位卡片，每张卡片显示地点名 + 一句秘辛预告（引流钩子）。
  状态标签：未探索 / 探索中 / 已勘验（盖印章）。
  顶部/底部提供 [随机探索] 按钮，系统随机抽取未探索地点作为目标。
  系统根据拥堵状态弱推荐一个冷门点（顶部轻量提示），但不强制。
  游客可自主选择任意未探索地点，点击后进入 Navigate。

Navigate · 路线引导与到达奖励：
  系统根据游客实时 GPS 定位，计算到目标地点的步行路线、预计时间、距离。
  若游客已在目标地点附近（GPS 距离 < 100m 或处于地理围栏内），提示"你似乎已在此地附近"。
  若目标为冷门点，文案包装为剧情引导（如"灵沼轩的地下图纸需要你去拼凑"）。
  底部操作区：
    [我已到达]（主按钮）→ 三件事同时发生：
      1. 后台标记该地点为"已勘验"，Explore 自动盖章；
      2. 弹出奖励小卡片（插画/徽章），停留 2-3 秒后自动收起；
      3. 自动解锁该地点完整秘辛（收纳在顶部导航"密档"入口）。
      随后自动进入 Narrative。
    [更换目标]（次要按钮）→ 放弃当前目标，返回 Explore 自选。
  设计意图：到达即奖励，三层反馈在一个动作里完成，不跳转独立 Reward 页面。

Narrative · 单点剧情 (AI 动态叙事)：
  到达后解锁该地点基于游客兴趣标签的单一叙事（title + baseContent + flavorText）。
  不设"调查工艺细节 / 审问 NPC"等多选项卡，叙事一次性完整呈现。
  顶部导航栏常驻"密档"按钮，游客可随时查看已解锁的秘辛收藏册（全部 6 张默认上锁）。
  底部操作区：
    [去下一个地点]（主按钮）→ 系统推荐下一目标并进入 Navigate。
    [查看其它视角]（次要按钮）→ 展开同地点不同兴趣标签的叙事卡片，增加停留与探索感。
  关键：Narrative 专注当前地点内容消费，换地点的决策放在 Navigate 阶段。

第三阶段：奖励与循环
循环触发：Narrative 底部 [去下一个地点] → Navigate → 重复 Explore-Navigate-Narrative 闭环。
取消独立 Reward 页面；奖励（盖章 + 弹卡 + 解锁秘辛）在 Navigate 的 [我已到达] 动作中一次性完成。

第四阶段：闭环结算与激励 (留存与变现)
勋章结算 (获得勋章)：剧情终点或游客主动结束游览时，系统根据全程表现（任务完成度、探索覆盖率）赋予数字勋章。
出园结算页 (待设计)：
  候选方向：秘辛收集册 / 路线足迹图 / 数字勋章墙 / AI 个性化结语。
  核心目标待确认：分享裂变 vs 情感闭合。
