二、 引导用户进行 AI 交互的策略 (提示逻辑)
为了让游客意识到“可以对话”，我们需要加入“动态引导机制”：
视觉暗示：
底部麦克风按钮周围增加一个缓慢跳动的金色光晕，模拟“监听中”或“待命”状态。
气泡引导：
在麦克风上方悬浮一个微小对话气泡，每隔 5 秒轮换一次文案，显示 AI 擅长回答的方向：
“问我关于这里的秘辛...”
“问我关于建筑的工艺...”
“问我有关历史的趣闻...”
状态反馈：
当用户长按麦克风时，金色按钮转变为波纹涟漪动效，文本叙事区显示“正在聆听您的疑问…”，将用户的长按操作与 AI 的处理状态通过视觉动效连接起来。
三、 技术实现的“避坑”逻辑
由于黑客松时间紧迫，语音交互的开发必须走“最简链路”：
Prompt 预注入 (重要)：
在调用 LLM 时，一定要在 System Prompt 中加入：
"你现在是故宫的数字秘辛讲述者。请始终保持神秘且克制的语气。无论游客询问什么，你的回复内容必须紧扣当前地点的地理背景和历史文化，回复字数控制在 80 字以内，确保易读性。"
交互闭环控制：
用户长按麦克风：前端调用 Web MediaRecorder API 捕获音频 -> 发送至 Whisper API。
Whisper 返回文本：前端显示“您问：[识别后的文字]”。
LLM 处理：后端将该问题连同当前的 Location_ID 和 Session_ID 发送给 LLM。
回复展示：AI 的回复不要直接覆盖旧文本，而是在叙事区下方以“金色引用框”的形式插入，形成明显的区分。
